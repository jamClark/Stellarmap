Item Generator: Firearm Inherited Features

These properties control how the weapon behaves when fired during combat. Take notice of the differences and similarities between this and the regular Weap Stats Tab. Remember, the Weapon Stats tab control how the firearm works if it is used as melee weapon by switching the attack mode. This tab controls the actual firing properties.

Max Damage

SetGunClass(int)

The maximum damage dealt by a single shot before being adjusted by skills, stats, combat environment, ammo, number of shots fired, accuracy of the weapon, or randomness.


Ammo Type

SetAmmoClass(string)

Identifies what types of ammo can be used in this weapon. Ammo that can be used will have the same value set for its Ammo Type as well.


Gun Damage Type

SetGunDamageType(int)

The same deal as regular Damage Types for a weapon, only this applies to the weapo being shot. See the Damage Type entry in Weapon Stats for more details.


Gun Size

SetGunSize(int)

Determines the size of the gun in relation to holsters. Holsters designed to hold guns will have a min and max range they can accept. If this object is outside that range then it won't fit in the holster.

  • GUNSIZE_NULL - n/a
  • GUNSIZE_TINY - deringer sized
  • GUNSIZE_SMALL - pistols, sub-machine guns, sawn-offs, maybe some tiny carbines
  • GUNSIZE_MEDIUM - most carbines, shotguns, many assult rifles
  • GUNSIZE_LARGE - some large assult rifles, sniper rifles
  • GUNSIZE_HUGE - mounted gun / or shoulder weapon (rocket launcher)
  • GUNSIZE_MASSIVE - think cannon
  • GUNSIZE_RIDICULOUS - ICBM missile maybe?
  • GUNSIZE_WTF - hell, I'm just being goofy now... Death Star I guess :D


Attachments

SetAcceptedAttachments(mapping)

Largely unused at this time. It allows you to specify what types of attachment this weapon takes. Technically there is a silencer and scope object but neither of them do anything at this time.


Firemodes

SetCapableFiremodes(int)

Sets a combination of firemodes that the weapon supports. Players can switch through the modes using the 'firemode' command. The first three modes fire 1, 2, and 3 shots per attack. The fourth mode by default is four but can be changed with User Firerate.


User Fireate

SetUserRate(int)

The rate of fire if a weapon is allowed to be switched into the fourth firemode known as 'user rate'.


Accuracy

SetAccuracy(int)

The percentage chance that the weapon will hit the target the player aimed for. This is not affected by player skill at all and is only used in combat calculations after all other values have been determined. Bascially, it is the accuracy of the weapon ie: the ability for it to put the bullet where the shooterr aimed.


Accuracy Dropoff

SetAccuracyDropoff(int)

The ammount of accuracy to loose for each successive shot fired when the firemode of the weapon is something other that single-shot. For example, if the firemode is FIREMODE_BURST with three shots per attack, and the starting accuracy is 100% and this value is 10 then each shot will be at 100%, 90%, and 80% accuracy respectively.


Round Based

SetMagIsRound(int)

Determines the arch-type of the weapon. If set the weapon uses ammo as a single round and stored those round internally one at a time. The weapon itself is responsible for tracking ammo count. Useful for things like shotguns, crossbows, and rocket launchers were each round must be manually loaded and no magazines are used. If not set the the ammo is treated as a magazine instead and the ammo count comes from the ammo object.


Max Rounds

SetMaxRounds(int)

Only used if Round Based is checked. This determines how many round can be loaded into the weapon. For example, a revolver might only let six be loaded.
