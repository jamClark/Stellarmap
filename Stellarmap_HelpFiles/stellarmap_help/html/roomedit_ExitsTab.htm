Room Editor: Exits Tab

The exits tab allows you to set the exits and enters of you room as well as assign doors to those exits. The enters list is enterly comprised of items from the static item list on the content tab of the room editor. The exits list contains everything the enterables list has plus several default values for common exits such as north, south, east, west, and so on. Just like inventories on the content page,, the doors list is propgated from the imported door objects in the workspace. You cannot assign a door until after an exit has been established. Take note that exits created in the map view domain show up in the exist list here. As well, you can manually edit the list here and the changes will be reflected in the map view. When you add a door to an exit, the map view will display a little yellow box around the exit node.

Enterables

SetEnters(mapping)

A collection of static 'objects' that the player can enter using the enter command. The left box contains all the elements created in the static items menu of the content tab. The right box is all of the rooms imported into the workspace.


Exits

SetExits(mapping)

The directions the player can exit. Consists of the static items from the content tab plus the default values north, south, east, west, northeast, northwest, souteast, southwest, up, down, and out. Assigning an exit to another room does not assign any exit from that desination back to the current room. Exits are effectively one-way unless you set them as otherwise.


Doors

SetDoors(mapping) Notice! This is not a part of default Dead Souls and must be inserted into the mud lib. Read more here.

The door attached to the exits or entrances of this room. Assigning a door to an exit does not cause that door to be assigned to any exit in the attached room.
