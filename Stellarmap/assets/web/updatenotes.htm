Release Notes

Because there are two versions of Stellarmap, one for general Dead Souls use and one with Stellarmass-specific features, some updates may apply only to one or the other. In such cases the symbols <DS> and <SM> will be used to signify which appropriately.

Back to Dowload Page


June 1, 2010

  • Mappings can now take arrays as a key. Finally, QCS-generated rooms can be imported without editing!
  • Mappings and arrays now perform some extra formatting checks to ensure any trailing commas are removed (the parser will choke on it). Unfortunately this means code that was nicely formatted by people or QCS will become somewhat mangled after being imported and saved in Stellarmap.
  • Fixed a bug that would not allow you to move rooms down and to the right when the viewport was scrolled too far above and to the left of the origin.
  • Probably a few other things over the last couple months that I can't remember now.

March 20, 2010

  • Changed the room properties box so that it uses a list defined in the configuration file
  • Added 'login' property for rooms
  • <SM> Added 'hospital' property for rooms
  • Added function controls for SetTerrainType() and SetMedium() for rooms
  • Function controls now have the ability to specify an additional header file they might require to be included.
  • Fixed a bug that was causing data to not save correctly
  • Fixed a bug with imported rooms clearing their data after selecting a different room (at least, I think I fixed it... that is to say.. it just went away magically after a couple weeks of not doing anything on this project)
  • Fixed another bug that would cause a crash if a room were removed from the project (deleted) and then re-imported back in without first saveing and reloading the workspace.
  • Fixed item generators so that they properly output their data. Whoops.

February 20, 2010

  • Integrated a spell checker into the basic text editor. Also added the standard cut/copy/paste context menu items.
  • text editor is now (approximately) 80 characters wide (getting it spot on is tougher than I thought it would be though)
  • fixed npc inventory crasher due to bad parameter type
  • fixed bad output for vendor storage room due to bad parameter type
  • external domains can now have their inventory and room names imported into a workspace. Just place the master .XML file in the /assets/domainreferences folder and they will show up on the import list
  • many internal refactorings of code will hopefully make life easier down the road
  • added access to text editor to all textbox-related function controls
  • Npcs can now be given points for other languages
  • <SM> Added flag to signal if an NPC should drop its inventory on death or just destroy it
  • <SM> Added a map of item drops rates on npc death
  • <SM> Added spacesuit option for armor
  • Ok, npc inventories work for real this time
  • Added one more encoding choice. A non-signature version of UTF8. Mostly for backwards compatibility. It has been set as the default encoding
  • fixed a crasher when canceling openning a domain for the first time
  • fixed a bug that was causing enters to be mapped to exits
  • fixed a bug that could cause a crash when assigning exits or enters to rooms just after loading a domain but before selecting any rooms that did not previously have exits or enters
  • fixed a bug where if a room had multiple exits going to the same desination all exits would be removed if any one was removed
  • changed item/armor generators' calls from 'SetValue()' to 'SetBaseCost()'
  • Added cost generators (however, prices are based on Stellarmass)
  • Renaming rooms is difficult and error prone so I took the lazy way out. There are now two ways to create a room. The old way (quick) where a name is assigned and the new way (slow) where you have to fill in a textbox with the new name
  • Added the ability to import items into the workspace. You can now reference them in the inventories lists
  • Added meals to the generator
  • Redesigned the config file to allow arbitrary lists to be added. This allows users to define their own custom lists within the config file that can then be accessed by the dynamic item generators. I've also appended many default lists from Dead Souls that weren't available before including: build types, bodytypes, mouth types, vision, resperation types, size types, meal types, mediums, terrain types, motion, and booby traps. See the config file for examples of how to add new types using the <list> tag
  • Added support for multi-parameter dynamic function controls. See inheriteditems.xml and the entry for meals for an example

February 2, 2010

  • Fixed a couple more show stoppers that appeared in the last release due to a bad config file for Sentient NPCs. Be sure to delete any old version of the file inheritednpcs.xml so that the new executable can create a new version without the errors in it.
  • Made the font bigger in the text editor windows for strings. I must be gettin' old

January 30, 2010

  • The location of the last project you opened or created is now saved in the config file
  • The currently selected room's name is now displayed above the Domain Map View rather than on the room edior panel
  • Door generator now creates an array of Ids rather than a single string. It also tries to create default ids for most common doors
  • Npcs' inventory lists now works
  • Added Sentient Npcs (LIB_SENTIENT), Vendors, Barkeeps to the NPC generator
  • Added support for text file encoding/decoding choice (found in Domain Options in the menu)
  • Added support for line ending choice (found in Domain Options in the menu)
  • Removed some of the features that were relevant only to our mud (firearms and ammo generators)
  • There are now two disinct binaries generated. One with Stellarmass-only features and one for general Dead Souls users
  • <SM> Added 'mechs allowed' property to rooms
  • <DS> Removed the native Windows API call to MessageBeep(). Hopefully this will make Stellarmap more Mono compatible
  • Found and fixed a couple show stoppers in the function controls used for processing mappings.
