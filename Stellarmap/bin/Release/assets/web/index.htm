STELLARMAP: The Visual Editor for DeadSouls Domains, Items, and Npcs!

Last Update: March ?, 2010

I dunno about you guys but I hate typing anything I don't have to. Dead souls is awesome in that I can custom code everything, and QCS is great for quick stuff but it gets messy when things get big. Remembering long lists of stuff or even which room I'm in when I edit can be a pain. For the less technical members of our mud its also a pain to explain LPC and trying to teach them QCS is almost as hard (maybe harder). So to that end I have create Stellarmap, a visual editor and code generator for Dead Souls domains, items, and npcs!
It ain't pretty and it ain't elegant but it gets the job done. The features include:
  • Visually create domain rooms and floor plans
  • Edit most common properties of rooms like, short descriptions, long descriptions, static items, room properties, etc
  • Visually connect rooms to one another to generate exits 
  • Generate exits and enters using menus
  • Fill rooms' inventories
  • Generate basic items, doors, armors and npcs - all of which are added to the domain workspace so that they can be included into the rooms using menus
  • Configurable lists - so each mud can have the proper races, classes, monies, limbs, etc for their mud when generating items and npcs
  • Cross-reference domains' rooms and items using automatically defined constants so that they can easily be shared without having to hardcode directories.
  • Extensible generators - if you have a special item or creature that inherits basic functionality from LIB_ITEM, LIB_ARMOR, or LIB_NPC it may be possible to edit config files to add those extensions to Stellarmap's generators! No recompiling, no mess!

 Known Issues:
I've been putting in a lot of late hours on this and have been rushing through the whole thing so it has only had a cursory glance for testing. Consider this beta software until it has been out and in use for a while. As of now, there are a few known issues to watch out for:


 General Issues:
  • removing static items from a room does not remove any smells/sounds/searches/exits/enters etc based on them. You will have to remove them manually
  • using the mouse wheel to scroll the domain map may also cause the main window to scroll

Function Controls Issues:

  • It's pretty easy to forget which controls automatically insert quotes around your text and which do not. Remember, the text you type gets translated literally to the code generated. In some cases you can fudge things by using the verbatim symbol '$' at the start. A good example is the SetAction, function for NPCs. The second parameter is automatically turned into a string literal unless you use $ in which case you can pass an array
  • There is no proper escape sequence for quotes. Having a \' at the end of a textbox can cause it to choke, try using the verbatim symbol and the encasing the entire string in quotes in such cases.
  • There is no proper escape for $ at the start of strings

Other Notes:
There are two very minor mods that must be made to Dead Souls for doors and armors to work. You can read more about it with the help file included. All you have to do is add a small and simple function to exits.c and base_armor.c. I tried to avoid this but really this turned out to the most effective option.
